# 邮件推送功能说明

## 功能概述

邮件推送功能支持创建、管理和执行批量邮件发送任务，主要特性包括：

- ✅ 支持增删改查邮件推送任务
- ✅ 支持系统内置发件人/收件人或手动输入
- ✅ 支持多种邮箱类型（QQ、163、126、Gmail、Outlook、Foxmail）
- ✅ 任务的启动和停止控制
- ✅ 实时查看邮件发送记录
- ✅ 失败邮件重新发送

## 数据库表结构

### 1. 邮件推送任务表 (t_email_push_task)

存储邮件推送任务的基本信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| task_name | varchar(100) | 推送任务名称 |
| email_title | varchar(200) | 邮件标题 |
| email_content | text | 邮件内容 |
| sender_type | int | 发件人类型：1-系统内置，2-手动输入 |
| sender_id | int | 发件人ID（系统内置时使用） |
| sender_email | varchar(100) | 发件人邮箱（手动输入时使用） |
| sender_password | varchar(100) | 发件人密码/授权码（手动输入时使用） |
| receiver_type | int | 收件人类型：1-系统内置，2-手动输入 |
| receiver_ids | text | 收件人ID列表（系统内置，逗号分隔） |
| receiver_emails | text | 收件人邮箱列表（手动输入，逗号分隔） |
| status | int | 任务状态：0-未开始，1-运行中，2-已完成，3-已停止，4-失败 |
| total_emails | int | 总邮件数 |
| success_count | int | 成功发送数 |
| fail_count | int | 失败发送数 |
| create_user_id | bigint | 创建人ID |
| create_user_name | varchar(50) | 创建人姓名 |
| create_time | datetime | 创建时间 |
| modify_time | datetime | 最后修改时间 |
| start_time | datetime | 开始时间 |
| end_time | datetime | 结束时间 |

### 2. 邮件发送记录表 (t_email_send_record)

存储每条邮件的发送记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键ID |
| task_id | bigint | 推送任务ID |
| sender_email | varchar(100) | 发件人邮箱 |
| receiver_email | varchar(100) | 收件人邮箱 |
| email_title | varchar(200) | 邮件标题 |
| email_content | text | 邮件内容 |
| send_status | int | 发送状态：0-待发送，1-成功，2-失败 |
| fail_reason | text | 失败原因 |
| retry_count | int | 重试次数 |
| send_time | datetime | 发送时间 |
| create_time | datetime | 创建时间 |
| modify_time | datetime | 最后修改时间 |

## API接口

### 邮件推送任务接口

#### 1. 分页查询推送任务
```
GET /api/push-task/page?pageNum=1&pageSize=10&taskName=xxx&status=0
```

#### 2. 查询任务详情
```
GET /api/push-task/{id}
```

#### 3. 创建推送任务
```
POST /api/push-task
Content-Type: application/json

{
  "taskName": "测试任务",
  "emailTitle": "测试邮件",
  "emailContent": "这是一封测试邮件",
  "senderType": 1,
  "senderId": 1,
  "receiverType": 1,
  "receiverIds": [1, 2, 3]
}
```

#### 4. 更新推送任务
```
PUT /api/push-task
Content-Type: application/json

{
  "id": 1,
  "taskName": "更新后的任务名称",
  "emailTitle": "更新后的邮件标题"
}
```

#### 5. 删除推送任务
```
DELETE /api/push-task/{id}
```

#### 6. 启动推送任务
```
POST /api/push-task/{id}/start
```

#### 7. 停止推送任务
```
POST /api/push-task/{id}/stop
```

### 邮件发送记录接口

#### 1. 分页查询发送记录
```
GET /api/send-record/page?pageNum=1&pageSize=10&taskId=1&sendStatus=2
```

#### 2. 查询发送记录详情
```
GET /api/send-record/{id}
```

#### 3. 重新发送邮件
```
POST /api/send-record/{id}/resend
```

## 支持的邮箱类型

系统支持以下邮箱类型的SMTP发送：

| 邮箱类型 | SMTP服务器 | 端口 | 加密方式 |
|---------|-----------|------|---------|
| QQ邮箱 | smtp.qq.com | 465 | SSL |
| 163邮箱 | smtp.163.com | 465 | SSL |
| 126邮箱 | smtp.126.com | 465 | SSL |
| Gmail | smtp.gmail.com | 587 | TLS |
| Outlook | smtp-mail.outlook.com | 587 | TLS |
| Foxmail | smtp.qq.com | 465 | SSL |

**注意：**
- QQ邮箱、163邮箱、126邮箱、Foxmail需要使用授权码，而非登录密码
- Gmail需要开启"不够安全的应用访问"或使用应用专用密码
- Outlook需要启用SMTP服务

## 前端使用说明

### 访问邮件推送页面

1. 登录系统后，点击左侧菜单的"邮件推送"按钮
2. 进入邮件推送管理页面

### 创建推送任务

1. 点击"新建任务"按钮
2. 填写任务信息：
   - 任务名称
   - 邮件标题
   - 邮件内容
3. 选择发件人类型：
   - **系统内置发件人**：从下拉列表选择已配置的发件人
   - **手动输入**：直接输入发件人邮箱和密码/授权码
4. 选择收件人类型：
   - **系统内置收件人**：从列表中多选收件人（按住Ctrl键）
   - **手动输入**：输入多个邮箱地址，用逗号分隔
5. 点击"保存"按钮

### 管理任务

在任务列表中，您可以：
- **启动任务**：点击"启动"按钮开始发送邮件
- **停止任务**：点击"停止"按钮暂停发送
- **查看详情**：点击"详情"按钮查看发送记录
- **编辑任务**：点击"编辑"按钮修改任务信息
- **删除任务**：点击"删除"按钮删除任务

### 查看发送记录

1. 点击任务的"详情"按钮
2. 查看每条邮件的发送状态
3. 对于失败的邮件，可以点击"重发"按钮重新发送

## 使用流程

### 1. 准备发件人和收件人（可选）

如果使用系统内置的发件人/收件人，需要先创建：

- 发件人管理：配置邮箱地址和授权码
- 收件人管理：添加收件人邮箱地址

### 2. 创建推送任务

调用创建任务接口，填写：
- 任务名称
- 邮件标题和内容
- 选择发件人（系统内置或手动输入）
- 选择收件人（系统内置或手动输入）

### 3. 启动任务

调用启动任务接口，系统会：
- 验证任务状态
- 获取收件人列表
- 创建发送记录
- 异步执行邮件发送

### 4. 监控任务进度

可以通过以下方式监控：
- 查询任务详情，查看成功/失败数量
- 查询发送记录，查看每条邮件的发送状态

### 5. 处理失败邮件

对于发送失败的邮件：
- 查看失败原因
- 点击重新发送按钮进行重试

## 技术实现

### 核心组件

1. **EmailPushTaskEntity** - 推送任务实体
2. **EmailSendRecordEntity** - 发送记录实体
3. **EmailService** - 邮件发送服务（支持多种邮箱类型）
4. **EmailPushTaskServiceImpl** - 任务管理服务（含异步执行）
5. **EmailSendRecordServiceImpl** - 记录管理和重发服务

### 异步执行

任务启动后使用 `CompletableFuture.runAsync()` 异步执行，避免阻塞主线程。

### 事务控制

关键操作使用 `@Transactional` 保证数据一致性。

## 注意事项

1. **授权码配置**：大部分邮箱服务商要求使用授权码而非登录密码
2. **发送频率限制**：注意邮箱服务商的发送频率限制，避免被封禁
3. **任务状态管理**：只有"未开始"或"已停止"的任务可以启动，只有"运行中"的任务可以停止
4. **重试机制**：失败邮件可以重新发送，系统会记录重试次数
5. **并发控制**：同一时间建议不要启动过多任务，避免资源竞争
